# SPEC.md — A Piece of Design

## Project Overview

**A Piece of Design** is a web application that serves as an AI-powered "bad design awards" platform. Users submit examples of poor design from any category (UI/UX, physical products, architecture, signage, packaging, etc.), and a multimodal AI automatically analyzes and "roasts" each submission with a structured Design Roast Report. The community then votes on submissions, and periodic awards ceremonies crown the worst designs.

---

## User Stories

### Submission Flow

- As a user, I want to upload a photo of a bad design with a short description so that I can share it with the community.
- As a user, I want to select a category tag (UI/UX, Physical Product, Architecture, Signage, Packaging, Other) when submitting so that submissions are organized and browsable.
- As a user, I want to receive an AI-generated "Design Roast Report" immediately after submitting so that I can see why my submission qualifies as bad design.

### AI Roast Report

- As a user, I want to see a "💩 Score" (1–10) for each submission so that I can quickly gauge how bad the design is.
- As a user, I want to see which of Nielsen's 10 Usability Heuristics the design violates so that I learn something from the experience.
- As a user, I want to read a witty roast paragraph generated by AI so that the experience is entertaining.
- As a user, I want to see a "How to Fix It" suggestion so that the platform is educational, not just mean.

### Browsing & Voting

- As a user, I want to browse a public gallery of approved submissions so that I can discover bad designs submitted by others.
- As a user, I want to filter the gallery by category, date, and 💩 Score so that I can find what interests me.
- As a user, I want to upvote or downvote submissions so that the community decides what rises to the top.
- As a user, I want to comment on submissions so that I can discuss why the design is bad or share similar experiences.

### Leaderboard & Awards

- As a user, I want to see a weekly and monthly leaderboard ranked by community votes so that I know the current "winners."
- As a user, I want to see tongue-in-cheek trophy badges (e.g., "The Golden Toilet Seat," "The Silver Facepalm") awarded to top-ranked submissions so that the experience feels like a real awards show.

### Hall of Shame

- As a user, I want to browse an all-time "Hall of Shame" archive so that I can see the historically worst designs.

### Authentication & Profile

- As a user, I want to sign up and log in via OAuth (Google/GitHub) so that I don't need to create a new password.
- As a user, I want a profile page showing my submissions, votes, and any trophies I've earned so that I can track my participation.

### Moderation

- As a moderator, I want to review AI-flagged submissions before they go public so that offensive or irrelevant content is filtered out.
- As a user, I want to flag inappropriate submissions so that the community stays respectful.

### Redesign Proposals (Stretch Goal)

- As a user, I want to submit a "Before/After" redesign proposal for an existing bad design so that I can show how it could be improved.

## Desired Specifications

### Functional Requirements

1. **Image Upload**: Accept JPEG, PNG, and WebP images up to 10MB.
2. **AI Analysis**: Use a multimodal AI API (implementation: Groq Llama 4 Scout vision, or equivalent) to generate the Design Roast Report within 15 seconds of submission.
3. **Voting System**: Each authenticated user gets one upvote or downvote per submission. Votes are toggleable.
4. **Leaderboard**: Auto-updated rankings with weekly and monthly periods. Resets at the start of each period but historical data is preserved.
5. **Gallery**: Paginated, filterable by category, sortable by date / 💩 Score / votes.
6. **Moderation Queue**: Submissions flagged by AI (low-confidence or potentially offensive) go to a moderation queue before appearing publicly.
7. **Responsive Design**: Fully functional on mobile and desktop.

### Non-Functional Requirements

1. **Performance**: Gallery page loads in under 2 seconds. AI report generation completes in under 15 seconds.
2. **Scalability**: Architecture supports up to 10,000 submissions and 50,000 votes without performance degradation.
3. **Security**: All user-uploaded images are scanned for malicious content. OAuth-only authentication (no password storage).
4. **Accessibility**: WCAG 2.1 AA compliance for core flows (submission, browsing, voting).

## GitHub Issues Breakdown

This project is decomposed into the following 7 issues:

1. **Project Setup & Infrastructure** — Initialize Next.js project, configure Supabase (database, auth, storage), set up deployment pipeline on Vercel, create DB schema.
2. **Authentication & User Profiles** — Implement OAuth login (Google/GitHub) via Supabase Auth, build user profile page showing submissions, votes, and trophies.
3. **Submission Flow & Image Upload** — Build the submission form (image upload + description + category tag), implement image validation and storage, create submission confirmation UI.
4. **AI Roast Report Generation** — Engineer prompts for multimodal AI analysis, integrate API (Groq vision + structured tool output), build the Roast Report display component (💩 Score, heuristics, roast, fix suggestion).
5. **Gallery & Filtering** — Build paginated public gallery, implement filters (category, date, score) and sorting, design submission detail page with full roast report.
6. **Voting System & Leaderboard** — Implement upvote/downvote logic, build weekly/monthly leaderboard with trophy badges, create Hall of Shame archive page.
7. **Moderation & Polish** — Build moderation queue for flagged submissions, add community flagging feature, responsive design pass, accessibility review, final deployment.
